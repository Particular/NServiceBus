NServiceBus

Welcome to NServiceBus. This package contains set of tools, code samples and documentation designed to help you develop and build NServiceBus applications.

For access to additional resources and information, such as downloads, forum posts, and extra documentation, go to the Particular website.

The NServiceBus package contains the following items:

  • Binaries - The NServiceBus binaries required to run NServiceBus.
  • Tools - A series of useful tools for management of messages.
  • Samples - Sample applications to get familiar on how to perform daily NServiceBus tasks.

Tools description:

  • TimeoutMigrator.exe
    This tool will extract your version 2.6 Timeouts and send them over to the new 3.0 TM that will be responsible for managing them.
  • ReturnToSourceQueue.exe
    This tool sends messages from the error queue back to its original queue so that they can be processed again.
  • XsdGenerator.exe
    Allows you to take your message schema classes and generate an XSD that will allow developers on other platforms to use to send messages with (though those messages would likely be coming in over HTTP via the gateway).
  • LicenseInstaller.exe
    Utility for developers to install a NServiceBus license file into the registry.

Samples description:

  • AsyncPages
    This sample illustrates the use of NServiceBus in a web application. What you'll see is how to configure NServiceBus for use in a web application, how to use the bus to send a message, and how to handle the response.
  • AsyncPages MVC3
    Similar to AsyncPages, this sample illustrates the use of NServiceBus in a ASP.NET MVC 3 web application. What you'll see is how to configure NServiceBus for use in a web application, how to use the bus to send a message, both in synchronous and asynchronous mode. and how to handle the response.
  • Encryption
    This sample illustrates how to encrypt data transmitted in messages by NServiceBus. What you'll see is how to define messages containing sensitive data, how to specify that the bus use an encryption service, and how to configure the keys that encryption service uses.
  • Full Duplex - Request/Response
    This sample illustrates how to full duplex - request/response communication works with NServiceBus. What you'll see is how to define messages, how to specify the destination for a message, and how to handle messages when they are received.
  • Generic Host - Logging
    This sample illustrates the various ways to use the NServiceBus host process to configure logging. You'll see the minimal amount of configuration necessary to set up logging, how to go back to standard log4net configuration, and the extensibility model using profiles.
  • PubSub
    This sample shows publish subscribe messaging using NServiceBus. This sample is also the "Getting Started" section of the documentation and shows polymorphic subscriptions as well as durable subscriptions for publisher fault-tolerance.
  • Versioning
    This sample is based on the PubSub sample above and shows how to go about versioning messages in this environment. You'll see how to inherit message types, how subscribers can continue to receive messages in the old schema even when the publisher publishes messages in the new schema.
  • Message Mutators
    Beginning NServiceBus V3, there are 2 types of message mutators, one that operates on the applicative messages and another one that operates on the physical transport messages. Reacting to individual messages can be used to perform things like validation of outgoing/incoming messages. Transport message works on the entire transport message are useful for things like compression, header manipulation etc. Find all about it in the sample or the the documentation about it.
  • Unobtrusive Mode
    Unobtrusive mode messages allows you to pass in your own conventions to determine which types are message definitions instead of using the IMessage interface. Other than the sample, you can read more about it here.
  • Data Bus
    Large chunks of data, such as images or video files can be transported using NServiceBus V3 data bus. The only thing you have to do is put an attribute over your large property and NServiceBus will take care of the rest. See how it works in the sample.
  • Scale out
    Similar in behavior to standard load balancers the NServiceBus Distributor is the key to scaling out message processing over many machines transparently. You can read more about it here, or see it in action in the sample.
  • Azure AsyncPages MVC3
    Similar to AsyncPages MVC3, but this time we use Windows Azure Cloud Services as a host and Windows Azure Queue Storage as a transport.
  • Azure Blob Storage Data Bus
    Similar to Data Bus, using Windows Azure Queue Storage as a transport and Windows Azure Blob Storage as a Data Bus implementation.
  • Azure Full Duplex
    Similar to Full Duplex, using Windows Azure Cloud Services as a host and Windows Azure Queue Storage as a transport.
  • Azure Host
    This sample demonstrates how you can use NServicebus and it's AsA_Host role to achieve high density hosting of worker roles on Windows Azure Cloud Services.
  • Azure PubSub
    Similar to Generic Host, but this time we use Windows Azure Cloud Services as a host, Windows Azure Queue Storage as a transport and Windows Azure Table Storage as a subscription persistence implementation.
  • Azure Sagas
    Similar to Starbucks, using Windows Azure Table Storage as a saga persistence implementation.
  • Azure Thumbnail Creator
    This sample shows you how all of the Windows Azure Queue Storage can be combined into a full application.
  • ServiceBus AsyncPages MVC3
    Similar to AsyncPages MVC3, but this time we use Windows Azure Cloud Services as a host and Windows Azure Service Bus Queues as a transport.
  • ServiceBus FullDuplex
    Similar to Full Duplex, using Windows Azure Cloud Serivces as a host and Windows Azure Service Bus Queues as a transport.
  • ServiceBus PubSub
    Similar to Generic Host, but this time we use Windows Azure Cloud Services as a host, Windows Azure Service Bus Queues as a transport and Windows Azure Table Storage as a subscription persistence implementation.
  • SQL Azure
    Similar to Starbucks, using Windows Azure Database as a saga persistence implementation.
  • TimeoutManager Listening On ServiceBus Queues
    Similar to Timeout Manager, using Windows Azure Service Bus Queues as a transport.
  • TimeoutManager Listening On Storage Queues
    Similar to Timeout Manager, using Windows Azure Queue Storage as a transport.
